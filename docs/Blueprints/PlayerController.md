## Overview

The Player Controller manages all player input, movement, and state transitions. In Unreal Engine 5, this is implemented primarily through the **Character Blueprint** with support from the **Animation Blueprint** for visual feedback.

## Implementation Notes

### Order of Implementation

1. **Basic Movement** (Walk, Run, Jump)
2. **Camera System** (Follow, collision)
3. **Sprint + Stamina**
4. **Dodge Roll**
5. **Ledge Detection + Climbing**
6. **Focus Mode**
7. **Animation Integration**

### Common Pitfalls

- **Don't forget** to set Character Movement component settings
- **Test** ledge detection on various geometry angles
- **Ensure** animation montages don't break state machine
- **Validate** stamina doesn't regenerate during sprint

### Blueprint Best Practices

- Use **Functions** for reusable logic
- Use **Macros** for common node patterns
- Keep **Event Graph** cleanâ€”delegate to functions
- **Comment** complex node clusters
- Use **Blueprint Interfaces** for interaction system
